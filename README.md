# Веб-приложение для управления и мониторинга симулятора БПЛА

Курсовой проект: **«Разработка веб-приложения для управления и мониторинга симулятора беспилотных авиационных систем»**.

## Назначение

Система предназначена для:

- ведения реестра беспилотных аппаратов (БПЛА);
- планирования полётных миссий с маршрутными точками;
- запуска, приостановки и остановки симуляции полёта;
- отображения телеметрии в реальном времени (WebSocket);
- визуализации траектории на карте;
- журналирования оповещений (низкий заряд, завершение миссии и т.д.);
- отображения погодных зон на карте и учёта погодных условий при расчёте маршрута.

## Запуск

### Требования

- JDK 17+
- Maven 3.8+
- **MySQL 8** (для профиля `mysql`) или ничего дополнительно (профиль `h2` по умолчанию)

### Вариант A — MySQL 

1. Запустите MySQL (Docker):

```bash
docker compose up -d
```

Создаётся БД `uas_db`, пользователь `uas` / пароль `uas`.

2. Запустите приложение с профилем `mysql`:

```bash
mvn spring-boot:run -Dspring-boot.run.profiles=mysql
```

В IntelliJ IDEA: **Run → Edit Configurations → Active profiles:** `mysql`

Схема создаётся автоматически **Flyway** из `src/main/resources/db/migration/V1__init_schema.sql`.

Переменные окружения (при необходимости):

| Переменная | По умолчанию |
|------------|--------------|
| `MYSQL_HOST` | localhost |
| `MYSQL_PORT` | 3306 |
| `MYSQL_DATABASE` | uas_db |
| `MYSQL_USER` | uas |
| `MYSQL_PASSWORD` | uas |

### Вариант B — H2 (быстрый старт без MySQL)

```bash
mvn spring-boot:run
```

Профиль `h2` включён по умолчанию. Данные в `./data/uas-db`.

Приложение: [http://localhost:8080](http://localhost:8080)

Консоль H2: [http://localhost:8080/h2-console](http://localhost:8080/h2-console)  
JDBC URL: `jdbc:h2:file:./data/uas-db`

Если ошибка «столбец не найден» — удалите папку `data/` и перезапустите.

### IntelliJ IDEA

1. Откройте проект как Maven-проект.

3. Для MySQL укажите профиль `mysql` (см. выше).

## Функциональные модули

### 1. Реестр БПЛА
CRUD через `GET/POST/DELETE /api/drones`. Хранятся позывной, модель, ограничения, координаты, заряд.

### 2. Миссии (сессии симуляции)
- На вкладке **Миссии** клик по карте **«Маршрут на карте»** добавляет точку маршрута (широта/долгота подставляются автоматически);
- высоту и скорость для каждой точки можно задать в полях справа от координат;
- `POST /api/sessions` — создание миссии с waypoints;
- `POST /api/sessions/{id}/start` — старт;
- `POST /api/sessions/{id}/pause` — пауза;
- `POST /api/sessions/{id}/stop?emergency=true` — аварийная остановка.

### 3. Движок симулятора (`SimulatorEngine`)
Периодически (500 мс) интерполирует положение между точками маршрута, обновляет телеметрию, расходует батарею, публикует данные в `/topic/telemetry`.

### 4. Мониторинг
Клиент подписывается на WebSocket и отображает:
- положение на карте;
- высоту, скорость, курс, крен, тангаж;
- текущие погодные условия в районе полёта;
- графики высоты и скорости.

### 5. Карта погоды и влияние условий
В интерфейсе маршрута и мониторинга отображаются погодные зоны: ветер, дождь, туман и гроза.
После расстановки точек маршрут анализируется не по одной стартовой координате, а по всей линии полёта: каждый участок между точками делится примерно на 200-метровые фрагменты, для каждого фрагмента определяется погода, после чего пересчитываются длина маршрута, примерное время полёта и погодная сводка. Цвет маршрута на карте также зависит от погоды на конкретном участке.

Погодные условия учитываются при расчёте маршрута и симуляции:
- снижают эффективную скорость БПЛА;
- уменьшают интенсивность разгона;
- увеличивают расход батареи;
- усиливают крен и тангаж в телеметрии;
- создают предупреждения при входе в опасную погодную зону.

Для локального профиля H2 сохранена папка `data/` с файлом базы, чтобы история миссий и накопленные записи не пропадали при запуске готового проекта.

## Сценарий демонстрации

1. Откройте **БПЛА** — в системе 3 демо-аппарата (UAV-ALPHA, UAV-BRAVO, UAV-CHARLIE).
2. Перейдите в **Миссии**, создайте маршрут (или используйте предзаполненные точки).
3. На **Панели** нажмите **Старт** (или выберите сессию в списке и запустите).
4. При составлении маршрута проверьте погодные зоны и примерное время полёта.
5. Откройте **Мониторинг** — наблюдайте полёт на карте, графиках и погодной карте.
6. Проверьте **Оповещения** при снижении заряда батареи и входе в сложные погодные условия.

## Роли пользователей и администрирование

В проект добавлена учебная ролевая модель без Spring Security, с хранением пользователей и действий в БД:

| Роль | Возможности |
|------|-------------|
| ADMIN | полный доступ, управление пользователями, просмотр и проверка журнала действий |
| OPERATOR | создание БПЛА, миссий, запуск/пауза/остановка, работа с оповещениями |
| OBSERVER | только просмотр панели, мониторинга, БПЛА, миссий и оповещений |

Тестовые пользователи создаются автоматически при первом запуске:

| Логин | Пароль | Роль |
|-------|--------|------|
| admin | admin123 | ADMIN |
| operator | operator123 | OPERATOR |
| observer | observer123 | OBSERVER |

Администратор видит отдельный раздел **«Админ»**, где можно менять роли, блокировать пользователей, задавать новый пароль и просматривать журнал действий пользователей.
